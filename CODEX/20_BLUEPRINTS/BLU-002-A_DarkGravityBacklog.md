# Agentic Command Center Backlog

This backlog was generated by the DarkGravity Architecture swarm based on the specifications in `BLU-002_PlaneForAgenticArchitecture.md`.

## Task 1: Set up Plane Cloud instance
Deploy a Plane Cloud instance for evaluation. Configure it with custom states matching the Agentic Development Lifecycle (Conversation, Spec Review, Sprint Active, Checkpoint, Verification, Merge Ready, Deployed, Cancelled). Create labels corresponding to the CODEX tag taxonomy and agent roles. Ensure the instance is accessible for testing the MCP bridge.
**Target Files:** `plane_cloud_config.yaml`

## Task 2: Audit Plane MCP Server
Thoroughly audit the `plane-mcp-server` GitHub repository to identify all exposed MCP tools. Document the capabilities, connection methods (Remote HTTP+OAuth, Remote HTTP+PAT, Local Stdio), and any limitations compared to the Plane REST API. This audit will determine if a fallback thin MCP adapter is necessary.
**Target Files:** `docs/plane_mcp_audit.md`

## Task 3: Integrate Plane MCP with IDE/Antigravity
Configure the IDE/Antigravity orchestrator to communicate with both the DarkGravity MCP server and the Plane MCP server simultaneously. Ensure the orchestrator can retrieve context from Plane (projects, cycles, items) and provide it to DarkGravity agents. Update the project brain (`.darkgravity.md`) with integration details.
**Target Files:** `.agent/context.md`, `antigravity/config.py`

## Task 4: Prototype Agentic Loop Flow
Implement and test a single, end-to-end Agentic Loop flow. The Architect creates an idea (Intake), a Work Item is generated, the DarkGravity pipeline (`coder`, `tester`) is triggered (Sprint Active), and the results are posted back to the Work Item (Verification) before merging.
**Target Files:** `scripts/plane_integration_prototype.py`

## Task 5: Implement Webhook Listener for State Changes
Set up a webhook listener within the DarkGravity ecosystem that triggers relevant pipelines based on Plane Work Item state changes. For example, transitioning an issue to "Sprint Active" should automatically invoke `darkgravity.run_pipeline` with the issue context. Ensure webhooks are secured with a shared secret.
**Target Files:** `darkgravity/webhooks.py`

## Task 6: Develop CODEX-Plane Synchronization Protocol
Define and implement a one-directional synchronization protocol from CODEX documents to Plane Pages. This ensures that the governance and architectural source of truth (CODEX) is automatically mirrored into Plane for organizational visibility and agent context.
**Target Files:** `scripts/codex_to_plane_sync.py`

## Task 7: Create Plane Views for Agent Dashboards
Create essential Plane Views for agents and the Architect, such as "My Active Sprints" (for individual agents), "Agent Activity" (for auditable trails), and "Blocked Items" (for human intervention). This provides the operational intelligence layer required by the Architect.
**Target Files:** `plane_views_config.json`
